总结的原理​

递归任务分解总结，或树状总结，从分段后的叶子节点开始总结，持续往根节点迭代总结，直到浓缩为一段关于这本书的总结。 为什么这么做，两方面原因：

  1. 由于人类阅读一本书需要很长时间，而让 LLM 进行分解总结可以很方便的直接从总结的文章摘要、章节摘要等阅读到最有价值的信息
  2. 另外由于 LLM tokens 限制，每次只能传入规定数量 tokens 文本，而无法把整本书一次性进行总结操作，需要分解总结。 参考：https://openai.com/research/summarizing-books
